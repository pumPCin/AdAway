Доход от рекламы

Существует часто высказываемое заблуждение: если пользователь никогда не кликает по рекламе, то её блокировка не наносит сайту или приложению финансового вреда. Это неверно, разработчики также зарабатывают деньги просто на показе рекламы. Блокировать рекламу или нет — решать вам. Лично я бы не стал пользоваться приложениями, которые показывают рекламу, так как считаю её очень раздражающей, поэтому без блокировщика я бы такие приложения просто не устанавливал.

Как работает AdAway?

AdAway использует файл hosts для блокировки доменных имён (хостов), которые распространяют рекламу. Файл hosts находится по пути /system/etc/hosts и сопоставляет имена узлов с IP-адресами. Это традиционный способ определения пар «имя узла — IP-адрес» без обращения к системе доменных имён (DNS). Все нежелательные хосты перенаправляются на адрес 127.0.0.1, что означает их перенаправление на ваше собственное устройство.
Если файл hosts недоступен для записи, в качестве альтернативного решения используется встроенная служба VPN. Она фильтрует соединения с нежелательными хостами, пропуская остальные.

Почему нужно перезагружать Android, чтобы изменения вступили в силу?

Java в Android поддерживает свой собственный внутренний кэш DNS. Операционная система отразит новый файл hosts немедленно (проверить это можно командой ping в командной строке), но вам потребуется перезагрузить Android, чтобы перестроить кэш DNS в Java.

Как использовать веб-сервер в AdAway?

AdAway будет работать и без использования веб-сервера!

Вы можете включить локальный веб-сервер в настройках AdAway, чтобы он отвечал на запросы по локальному IP-адресу 127.0.0.1. Это означает, что на запросы приложений к рекламным серверам, перенаправленные на 127.0.0.1, будет отвечать веб-сервер AdAway.
Некоторые приложения отказываются работать, когда рекламные серверы недоступны. С помощью этого метода они снова станут доступны, получая в ответ пустую страницу вместо рекламных изображений.

Как я могу заблокировать/разблокировать определённые хосты?

Добавьте имена узлов, которые вы хотите заблокировать, в список Заблокированные на главном экране. Кроме того, имена узлов, которые вы хотите исключить из блокировки, можно добавить в список Разрешённые, а имена, которые нужно перенаправить на конкретный IP-адрес — в список Перенаправленные.
